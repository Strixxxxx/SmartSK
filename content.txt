2. Non-Functional Requirements

The non-functional requirements define the performance, security, usability, and reliability standards of SmartSK.

2.1 The system must be accessible through any modern web browser (Chrome, Edge, Firefox, Safari).
2.2 The system must have a responsive user interface, developed using React + Vite + TypeScript and CSS modules.
2.3 The system must maintain 99.9% uptime, supported by reliable hosting and MSSQL database management.
2.4 The system must process and return analytics data within a minute for forecasting and predictive functions.
2.5 The system must encrypt sensitive data using AES-256 and secure connections via TLS/HTTPS.
2.6 The system must log all activities and store them securely for monitoring and auditing.
2.7 The system must support scalability up to 1,000 concurrent users via optimized Node.js + Express.js backend operations.
2.8 The system must ensure data consistency and integrity during concurrent transactions.
2.9 The system must allow the interface to be learnable within 10 minutes, prioritizing ease of use.
2.10 The system must automatically back up project data daily and restore within 15 minutes in case of system failure.

3. Design of Software, System, Product, and/or Processes

The design of SmartSK: A Web-Based Project Monitoring System with Budget Forecasting and Predictive Project Analysis follows a structured and modular development process to ensure efficiency, reliability, and data-driven decision-making for Sangguniang Kabataan officials.

3.1 Frontend: Built using React + Vite + TypeScript for responsive and modular interface development.
3.2 Backend: Developed with Node.js + Express.js, providing RESTful APIs for seamless communication.
3.3 Database: Implemented with Microsoft SQL Server (MSSQL) to ensure secure, scalable, and organized data storage.
3.4 AI and Forecasting: Powered by Python libraries with integrated with Gemini AI for predictive insights & Budget forecasting.
3.5 Security: Managed using bcrypt and crypto for password hashing, JWT for authentication, and Nodemailer for email/OTP verification.
3.6 Utilities: Includes dotenv for secure configuration, Multer for file uploads, fs and uuid for system processes, and Chokidar for automated file conversions.
3.7 Visualization: Forecasting and analysis outputs are presented through interactive dashboards and charts for intuitive data interpretation.

This architecture ensures SmartSK delivers a real-time, secure, and intelligent project monitoring environment that promotes transparency, accountability, and collaborative youth governance.
