The Root Cause: Your Azure Health Check is configured to ping the path /health. While your code defines this route, it's possible that Azure's internal networking or some other configuration is preventing access to non-/api prefixed routes. Since all your working routes use the /api prefix, the most reliable solution is to make the health check consistent with the rest of your application's API structure. Step 1: Update the Route in main.js

JavaScript

// Health Check Endpoint

JavaScript
