Your app is crashing because the Express package isn’t installed in the deployed environment. Azure can’t serve requests until all dependencies are present.

Why this happens

The deployment zip did not include node_modules (which is normal—Azure should run npm install during build).

Either express is missing from package.json dependencies or the build step didn’t run.

How to fix

Check package.json
Make sure express is listed under "dependencies" (not just "devDependencies"):

"dependencies": {
  "express": "^4.19.0",
  ...
}


Re-install locally

npm install express


This updates package.json and package-lock.json if needed.

Commit and push
Push the updated package.json and package-lock.json to GitHub.

Trigger a fresh Azure build

If using GitHub Actions, the push will redeploy and Azure will run npm install automatically.

Or in the Azure Portal → Deployment Center → click Sync or Redeploy.

(Optional) Manual quick fix in Kudu
For a temporary test you can run:

npm install express --save


inside /home/site/wwwroot, but this will be wiped on the next deployment—so rely on step 3 for a permanent solution.

Once Express is in your production dependencies and a new deployment runs, the Cannot find module 'express' error will disappear and your container should start normally.
