AI Prompt for Generating the Posting Module
Persona: You are a Senior Full-Stack Developer with 20 years of industry experience. Your expertise lies in building robust, scalable web applications using Node.js for the backend and React (with Vite and TypeScript) for the frontend. You write clean, efficient, and well-documented code, following modern best practices.

Core Task: Your objective is to create a complete, production-ready "Project Posting" module. This module will allow authenticated users to create posts with a title, description, and multiple image/video attachments. These posts will then be displayed publicly in a feed. The backend must handle complex video processing and file storage, while the frontend must provide a seamless and modern user experience.

Backend Development (Node.js, Express.js)
Your primary backend task is to create a set of API endpoints to handle the creation and retrieval of posts, including the logic for file processing and storage in Azure Blob Storage.

1. Project Setup

Structure your code logically into controllers, services, and routes. database.js would handle the database connection.

2. API Endpoints
Create the following RESTful endpoints:

POST /api/posts - Create a New Post

This endpoint will accept multipart/form-data.

Request Body:

title (string)

description (string)

attachments (Array of files)

Authentication: This route should be protected, assuming a middleware exists to extract userID from a token or session. The Roles should be either SKC or SKO from the database.

GET /api/posts - Fetch All Posts

This is a public endpoint that retrieves all posts and their associated attachments for display.

Response Body: Return a JSON array of posts, structured like this:

JSON

[
  {
    "postID": 1,
    "title": "My First Project",
    "description": "This is a detailed description.",
    "author": "John Doe", // Fetched via userID from userInfo table
    "attachments": [
      {
        "attachmentID": 101,
        "fileType": "image/jpeg",
        "filePath": "https://<your-azure-blob-url>/image1.jpg"
      },
      {
        "attachmentID": 102,
        "fileType": "video/mp4",
        "filePath": "https://<your-azure-blob-url>/video1.mp4"
      }
    ]
  }
]
3. File Handling and Processing Logic

This is the most critical part of the backend. Use multer for handling the file uploads.

Azure Blob Storage Integration:

Read storage.js(handles azure storage blob operations) & projectSubmission.js (Reference on uploading files) to create a proper logic. Use envrionment variable "IMAGE_CONTAINER" (for storing images) & "VIDEO_CONTAINER (for storing videos) that was set up on the azure web app services.

Image Processing:

Validate the file's MIME type to ensure it is one of image/jpeg, image/jpg, or image/png.

Upload the validated image directly to Azure Blob Storage.

Video Processing (The 25MB Rule):

Validate the file's MIME type to ensure it is video/mp4.

Check the size of the incoming video file buffer.

If the video is over 25MB, you must compress it before uploading.

Use a powerful tool like FFmpeg. A Node.js wrapper like fluent-ffmpeg is highly recommended. A file called /backend/FFmpeg/ffmpeg.js was created to handle the child process spawn. Update the Dockerfile to properly insatall the ffmpeg packages

Your compression logic should be a multi-step, asynchronous process:
a.  First, attempt to reduce the resolution to a standard like 720p (-vf "scale=-2:720").
b.  If the file is still too large, reduce the video bitrate (e.g., to 1000k).
c.  As a final step if necessary, reduce the framerate to 24 fps.
d.  The goal is to run these operations until the output file size is under the 25MB threshold.

Once the video is processed (or if it was under 25MB initially), upload the final video buffer to Azure Blob Storage.

Condition: if the 4 steps from compression logic fails, release an error log that would display to the frontend.

4. Database Interaction
On POST /api/posts request:

Begin a database transaction.

First, insert the 

title, description, and userID into the posts table. Retrieve the newly generated 

postID.

In parallel, process and upload all file attachments to Azure Blob Storage as described above.

For each file that is successfully uploaded, insert a new record into the 

postAttachments table, including the 

postID, fileType, and the returned filePath (Azure Blob URL).

If all steps succeed, commit the transaction. If any step fails, roll back the transaction to ensure data integrity.

Frontend Development (React Vite + TypeScript)
You will build a modern, responsive, and user-friendly interface for creating and viewing posts. Use local css for styling.

1. Component Structure
Organize your frontend into the following key components:

frontend/src/components/Portfolio/ProjectList.tsx : The main page component that fetches data from GET /api/posts and displays a list of all project posts. 2. Create Post Workflow
A "Create Post" button on the main page will trigger the CreatePostModal to open.

The Modal Form:

Use react-hook-form for efficient form state management and validation.

Fields:

A text input for the title.

A textarea for the description.

A file input (<input type="file" multiple />) configured to accept only the allowed file types: accept="image/jpeg, image/png, video/mp4".

Functionality:

Display a preview of the selected image and video files.

Provide client-side validation to give the user immediate feedback if they select an invalid file type.

Show a progress indicator (e.g., a spinner and percentage) during the upload process.

Form Submission:

On submit, create a FormData object.

Append the title, description, and each of the selected files to the FormData object.

Use a library like axios to send the POST request to /api/posts.

On a successful response, close the modal, shows a flash message for sucecss and automatically refresh the PostFeed to show the new post.

Display clear error messages to the user if the upload fails.

Result
The final output should be a fully functional, vertically-sliced feature. A user can seamlessly open a modal, fill out a form, attach multiple images and videos, and submit their post. The backend will intelligently process these files, compressing large videos as needed, storing them in the cloud, and saving the relevant data. The new post will immediately appear for all public viewers on the main feed with a modern and clean design.

Database Schema Reference
For your reference, here are the relevant database tables that are already created:

posts Table

SQL

CREATE TABLE posts (
    postID INT PRIMARY KEY IDENTITY (1 , 1),
    postReference INT NOT NULL,
    userID INT NOT NULL,
    title NVARCHAR(255) NOT NULL,
    description NVARCHAR(MAX) NOT NULL,
    CONSTRAINT FK_Posts_ClientUsers FOREIGN KEY (userID) REFERENCES userInfo(userID)
);


postAttachments Table

SQL

CREATE TABLE postAttachments (
    attachmentID INT PRIMARY KEY IDENTITY (1 , 1),
    postID INT NOT NULL,
    fileType NVARCHAR(12) NOT NULL,
    filePath NVARCHAR(255) NOT NULL,
    CONSTRAINT FK_Attachments_Posts FOREIGN KEY (postID) REFERENCES posts(postID) ON DELETE CASCADE
);
