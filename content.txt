Recommended startup.sh
#!/bin/bash
# Custom startup for Azure App Service (Node + Python)

echo "--- Custom Startup Script Started ---"

# Make the script fail only on critical errors, not on pip warnings
set -o errexit   # exit on serious errors
set -o nounset   # exit on undefined vars
set -o pipefail  # catch pipe errors

# Helper to log but not abort on a non-zero exit code
run_or_warn() {
    "$@" || echo "[WARN] Command failed (ignored): $*"
}

# Check that Python3 exists
if ! command -v python3 &>/dev/null; then
    echo "[FATAL] python3 not found. Exiting."
    exit 1
fi
echo "Python version: $(python3 --version)"

# Add user-local bin to PATH so `pip --user` installs are visible
export PATH="$HOME/.local/bin:$PATH"

# Ensure pip is present
if ! python3 -m pip --version &>/dev/null; then
    echo "pip not found, installing..."
    run_or_warn curl -sS https://bootstrap.pypa.io/get-pip.py | python3 - --user
fi

echo "Upgrading pip (non-fatal if already latest)…"
run_or_warn python3 -m pip install --upgrade pip --user --quiet

# Install dependencies if requirements.txt exists
if [ -f "requirements.txt" ]; then
    echo "Installing Python dependencies from requirements.txt…"
    run_or_warn python3 -m pip install -r requirements.txt --user --quiet
else
    echo "[WARN] requirements.txt not found, skipping Python dependencies."
fi

# Show Python path for debugging
run_or_warn python3 -c "import sys; print('Python exec:', sys.executable); print('Python path:', sys.path)"

# Finally start the Node.js app in the foreground
echo "Starting Node.js application…"
npm start

echo "--- Custom Startup Script Finished ---"

Why this works

run_or_warn wraps pip commands so a non-zero exit code prints a warning but does not stop the script.

We still exit 1 if Python itself is missing—because that’s a real blocker.

The Node process is launched in the foreground, keeping the container alive for Azure health checks.
