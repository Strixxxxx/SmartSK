AI Prompt: Create a Full-Stack Archiving Module
Your Role: You are an expert senior full-stack developer specializing in Node.js and React.

Situation
You are assigned to the "SmartSK" project, an existing project monitoring system. To improve performance and data management, the system requires a new module to handle the archiving of user accounts and projects. This will move inactive records from the primary tables to corresponding archive tables, keeping the main application responsive while preserving data integrity. The database schema and stored procedures for these actions have already been created and tested.

Task
Your task is to build the complete frontend and backend logic for this archiving module. The module must allow an authorized administrator to view lists of archived accounts and projects, and restore them to the active database tables on demand.

Technology Stack:

Backend: Node.js with Express.js

Database: Microsoft SQL Server (MSSQL)

Frontend: React Vite with TypeScript

Styling: Local CSS Modules (.module.css)

Action
You will implement the solution based on the predefined file structure and database logic. Ensure your code is clean, efficient, and includes proper error handling.

1. Backend Development (Node.js / Express)
You have two files to work in: accArchive.js and projArchive.js. These will contain the API routes. Assume a database connection file (db.js) that exports a configured mssql connection pool already exists.

Database Context:

Target Stored Procedures: You must use the following existing stored procedures to interact with the database. Do not write raw INSERT or DELETE queries for the archiving/restoring logic.


accArchived: Moves a user from userInfo to userInfoARC. 


prjArchived: Moves a project from projects to projectsARC. 


accReturn: Moves a user from userInfoARC back to userInfo. 


prjReturn: Moves a project from projectsARC back to projects. 

accArchive.js - Account Archive Routes:


GET /api/archive/accounts: Create an endpoint to fetch all records from the userInfoARC table.  Return the list as a JSON array.

POST /api/archive/accounts/restore/:userId: Create an endpoint that takes a userId from the URL parameter. Execute the accReturn stored procedure with the provided userId. Return a success message upon completion.

projArchive.js - Project Archive Routes:


GET /api/archive/projects: Create an endpoint to fetch all records from the projectsARC table.  Return the list as a JSON array.

POST /api/archive/projects/restore/:projectId: Create an endpoint that takes a projectId from the URL parameter. Execute the prjReturn stored procedure with the provided projectId. Return a success message upon completion.

General Backend Requirements:

Wrap all database calls in try...catch blocks.

Send meaningful JSON responses (e.g., { success: true, data: [...] } or { success: false, message: 'Error details' }).

Ensure the main server file (index.js or app.js) imports and uses these route files.

2. Frontend Development (React Vite + TypeScript)
You will create three components: Archive.tsx (the main container), AccArchive.tsx (for accounts), and PrjArchive.tsx (for projects), along with a corresponding CSS module for styling.

Archive.tsx - Main Module Component:

This component will serve as the main view for the archiving module.

Implement a state using useState to manage the currently active tab (e.g., 'accounts' or 'projects').

Create a simple tab navigation UI with two buttons: "Archived Accounts" and "Archived Projects". Clicking a button updates the active tab state.

Conditionally render the AccArchive component when the 'accounts' tab is active, and the PrjArchive component when the 'projects' tab is active.

AccArchive.tsx - Archived Accounts Sub-Module:

Use the useEffect hook to fetch the list of archived accounts from your backend endpoint (GET /api/archive/accounts) when the component mounts.

Manage component state for loading, error, and the array of archived accounts (archivedAccounts).

Display the fetched data in a clear, tabular format. Include columns for Full Name, Username, Email, Phone Number, and an Actions column.

In the Actions column of each row, add a "Restore" button.

When the "Restore" button is clicked, show a confirmation dialog (window.confirm()) to prevent accidental actions.

If the user confirms, call the backend endpoint (POST /api/archive/accounts/restore/:userId) with the correct userId.

After a successful restoration, remove the restored item from the local state to update the UI instantly and provide user feedback (e.g., a toast notification or simple alert).

PrjArchive.tsx - Archived Projects Sub-Module:

Follow the exact same pattern as AccArchive.tsx.

Fetch data from GET /api/archive/projects.

Display project data in a table. Include columns for Reference Number, Title, Submitted Date, and an Actions column.

The "Restore" button should call POST /api/archive/projects/restore/:projectId.

Implement the same confirmation, state management, and UI update logic.

Archive.module.css - Styling:

Create professional and clean styling for the module.

Style the tab buttons to clearly indicate which one is active.

Style the tables with appropriate padding, borders, and hover effects on rows.

Style the "Restore" buttons to be easily identifiable (e.g., with a distinct color).

Result
The final output will be a fully functional, production-ready archiving module that integrates seamlessly into the existing SmartSK system. The backend will provide secure and efficient API endpoints that leverage the pre-built stored procedures. The frontend will feature a user-friendly, tabbed interface that allows administrators to easily view and restore archived accounts and projects, with clear feedback and state management. The code across the stack should be well-documented, maintainable, and adhere to industry best practices.
