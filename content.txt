Here's why this might be happening and what you should check next:

Port Configuration: Azure App Service expects your application to listen on the port specified by the PORT environment variable, which is 3000 in your case as per the docker run command in the log. You need to ensure your Node.js application is configured to use process.env.PORT to set its listening port. Many frameworks (like Express.js) do this by default, but if you have hardcoded the port (e.g., app.listen(8080)), it won't work.

Middleware container: The logs show smartsk-backend_0_ebea6a8c_middleware. Azure App Service may use a separate middleware container for features like CORS or authentication. To fix this, you should:

Check your application's index.js or app.js file. Make sure the server is listening on the PORT environment variable. It should look something like this:

JavaScript

const port = process.env.PORT || 3000;
app.listen(port, () => {
  console.log(`Server is running on port ${port}`);
});
This ensures that when your application is deployed to Azure, it will listen on the port that Azure specifies.

Verify the start script in your package.json file in the backend folder. The start script is what the container will use to launch your application. A common issue is the start command not correctly pointing to your main file. For example, if your main file is named main.js inside the backend folder, your start script should be node backend/main.js or node main.js depending on the working directory context set in your deployment.
