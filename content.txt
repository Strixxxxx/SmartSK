  The goal is to reconstruct the project review system by introducing an automated AI process that analyzes submitted project proposals against a set of rules defined by the SK Chairperson. This
  plan incorporates your feedback for better code organization and dynamic, barangay-specific file naming.

  Phase 2: Backend Implementation (Node.js & Python)

   1. Environment Setup:
       * Add the new environment variable to the .env file in the backend directory:
   1         AIPROJ_CONTAINER="aiproj-rules"

   2. Create the AI Python Script (`projectAIJobs.py`):
       * Create a new file: backend/AI/projectAIJobs.py.
       * This script will:
           * Accept a projectID as a command-line argument.
           * Connect to the database to get the project's file_path and the submitter's barangayName.
           * Download the project document (PDF/DOCX) and the dynamically named rules file (e.g., PROJECT RULES - [barangayName].txt) from Azure Blob Storage.
           * Use libraries like pypdf and python-docx to extract text content.
           * Send the extracted text and rules to the Gemini API for analysis.
           * Save the AI's report and decision into the projectAudit table.

   3. Update Project Submission Logic:
       * Modify backend/projectSubmission/projectSubmission.js to use child_process.spawn to execute projectAIJobs.py asynchronously after a project is successfully submitted.

   4. Create New Project Audit API (`projectAudit.js`):
       * Create a new file: backend/projectaudit/projectAudit.js.
       * This file will contain all new endpoints for the AI project audit process (/api/projectaudit/ai-rules [GET, POST], /api/projectaudit/audit [GET], and /api/projectaudit/manual-override
         [POST]).

   5. Register New Route in `main.js`:
       * Modify backend/main.js to import and use the new projectAudit.js router under the /api/projectaudit path.
