The SmartSK Project Monitoring System is deployed using a hybrid cloud infrastructure that combines Microsoft Azure and Railway.com, designed for scalability, performance, and maintainability. The deployment leverages containerized services, automated CI/CD pipelines, and secure HTTPS endpoints to ensure efficient operation and minimal downtime. The backend, developed with Node.js and Express.js, integrates Python forecasting & predictive analysis module using Google Gemini API, while the frontend is a React Single Page Application (SPA) compiled with Vite and TypeScript.

The deployment architecture begins with centralized source control via a GitHub repository, where developers commit all updates, code changes, and patches. Every commit to the main branch automatically triggers the GitHub Actions CI/CD workflow, initiating the build and deployment of the backend container. The backend is packaged into a Docker image and pushed to Azure Container Registry (ACR), from which it is deployed to Azure App Service (Basic B1) under the container name smartsk-backend-container. This container environment includes both Node.js for API handling and Python runtime for executing AI-driven forecasting and predictive analytics.

Simultaneously, the frontend undergoes continuous deployment through Railway.com Auto-Deploy. Whenever changes are pushed to GitHub, Railway automatically builds and redeploys the React application. The frontend communicates with the Azure-hosted backend API through secure HTTPS endpoints, ensuring encrypted data transmission and synchronized updates between the two environments.

Azure resources were provisioned under the Southeast Asia region to reduce latency for the target users located in District 5, Quezon City. The Azure App Service provides a public endpoint for API access, while the Azure Container Registry securely manages backend container versions. Environment variables were configured on both the frontend and backend to manage API base URLs, JWT authentication keys, database connection strings, and AI model paths, all stored securely using Azure’s environment configuration tools.

The CI/CD pipeline ensures consistent integration and delivery. Each backend build uses GitHub Actions to automatically publish Docker containers to ACR and trigger redeployment to Azure App Service. The frontend follows a similar process on Railway, where every commit triggers a build, runs pre-deployment checks, and publishes the latest version to production. This automation guarantees that both layers of the system remain in sync after each iteration.

Before production rollout, database initialization and integration testing were conducted to verify communication between the frontend, backend, and database. MSSQL served as the main database, handling user data, project proposals, forecasting records, and predictive analysis results. Testing also included verification of RESTful API endpoints, AI model outputs, and dashboard rendering on various browsers. User Acceptance Testing (UAT) was performed by SK officials and administrators, confirming that the system met all specified requirements and achieved satisfactory usability.

The deployment was validated through several levels of testing, starting with smoke tests for endpoint health checks, followed by end-to-end testing of proposal submission, approval workflows, report generation, and forecasting visualizations. Once verified, the production instance was released for operational use, enabling real-time project tracking and data-driven decision-making for SK officials.

Post-deployment monitoring is handled through Azure Monitor and Railway project insights, which provide continuous performance metrics, error logs, and uptime tracking. Regular maintenance tasks such as container updates, dependency management, and AI model retraining are scheduled through GitHub workflows. In the event of deployment issues, rollback procedures are available through stored container artifacts in ACR and GitHub release management.

The infrastructure supporting the SmartSK deployment includes Azure App Service for backend API hosting, Azure Container Registry for container management, MSSQL for database operations, and Railway.com for frontend hosting. This hybrid architecture ensures that the system maintains high availability, low latency, and data integrity, while also allowing flexible scaling and future feature integration.

From a project management perspective, the implementation was overseen by designated team roles: the Lead Developer managed backend containerization and CI/CD automation, the Frontend Developer maintained interface deployment through Railway, the System Analyst handled API integration and testing coordination, the QA Tester validated functionality and stability, and the Project Manager supervised release cycles, feedback collection, and version control.

Overall, the deployment of SmartSK demonstrates a robust and modern implementation strategy that combines automation, scalability, and security. Through Azure cloud hosting, Railway frontend delivery, and GitHub-based CI/CD pipelines, the system achieves a streamlined, secure, and reliable production workflow — ensuring continuous service for SK officials and users while supporting future scalability and intelligent feature expansion.
