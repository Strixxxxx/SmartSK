  Here is the full, revised, and final plan:

  Phase 1: Database and Data Storage

   1. Database Migration: I will create a Query.txt file with the SQL script for the new rawDataUpTrack table to log CSV uploads. You will execute this script manually.
   2. Azure Storage: The system will use two pre-configured Azure Blob Storage containers: HA_CONTAINER for raw CSVs and JSON_CONTAINER for the generated reports.

  Phase 2: Backend Refactoring

   1. Asynchronous AI Engine (`aiJobs.py`): I will create a new, standalone Python script to run hourly.
       * Data Sourcing: It will primarily use CSVs from HA_CONTAINER. The fallback will be to execute the [Raw Data] stored procedure in the SQL database. No external data will be used.
       * Report Generation: It will generate three JSON reports: forecast.json, pa_analysis.json, and pa_trends.json.
       * Report Storage: It will atomically upload these reports to JSON_CONTAINER.

   2. Data Upload Endpoint (`rawData.js`): I will rewrite the file upload logic to perform a transactional "dual write" to both the SQL database and HA_CONTAINER.

   3. Scheduler (`main.js`): I will add a node-cron job to execute aiJobs.py hourly.

   4. Report Retrieval API (`AIDataRetrieval/`): I will create new API endpoints (/api/reports/forecast, etc.) that retrieve and serve the pre-generated JSON reports from JSON_CONTAINER.

   5. Cleanup: The obsolete pyBridgeFC.js and pyBridgePA.js files will be deleted.

  Phase 3: Frontend Implementation (Detailed)

   2. Predictive Analysis Page (`/frontend/src/components/Client/PredictiveAnalysis/`):
       * Core Change: All on-demand analysis is removed. The page will fetch two reports on load (pa_analysis.json and pa_trends.json). All filters will now operate on this client-side data for an
