✦ That is an excellent question, as it gets to the core of the access control logic. Here is how I understand and plan to implement the visibility options based on your prompt.

  When you select "Viewable to Barangay Only"

  This is the most restrictive option for authenticated users.

   * What the User Sees: When another user views your post, they will only see the tagged project details (or the encrypted attachments) if they are logged in AND they belong to the exact same 
     barangay as you (the author of the post).
   * How the System Enforces This (Backend Logic):
       1. When a logged-in user requests to view posts, the backend will know their userID and their associated barangayID from their authentication token.
       2. For your post, the system will retrieve the viewOption settings from the database.
       3. It will see that the opforBrgyProj (or opforBrgyEAttach) flag is set to true.
       4. The backend will then perform a check: Does the barangayID of the user viewing the post match the barangayID of you, the post's author?
       5. Only if the barangay IDs match will the server include the sensitive project/attachment details in the data sent back to the frontend. For everyone else (including users from other
          barangays and the general public), those details will be omitted.

  Example: You are from San Bartolome. You create a post and set project visibility to "Barangay Only".
   * A logged-in user from San Bartolome WILL see the tagged projects.
   * A logged-in user from Nagkaisang Nayon WILL NOT see the tagged projects.
   * A public user who is not logged in WILL NOT see the tagged projects.

  When you select "Viewable to All Barangays"

  This option acts as a gate for all authenticated users, regardless of their specific barangay.

   * What the User Sees: Anyone who is logged in to the system will be able to see the tagged project details or attachments. It doesn't matter which barangay they are from. Public, unauthenticated
     users will not see this information.
   * How the System Enforces This (Backend Logic):
       1. When a request is made, the system first checks if the user is authenticated. If not, the sensitive data is automatically hidden.
       2. If the user is authenticated, the system retrieves the viewOption settings for your post.
       3. It will see that the opforAllBrgyProj (or opforAllBrgyEAttach) flag is true.
       4. Because this flag is true, the system will include the project/attachment details in the response without needing to compare the viewer's barangay to your own. The only requirement is that
          they are a valid, logged-in user.

  Example: You are from San Bartolome. You create a post and set project visibility to "All Barangays".
   * A logged-in user from San Bartolome WILL see the tagged projects.
   * A logged-in user from Nagkaisang Nayon WILL also see the tagged projects.
   * A public user who is not logged in WILL NOT see the tagged projects.

  In summary, the system will use the user's authentication status and their associated barangayID in combination with the viewOption flags you set for each post to conditionally serve the
  sensitive data, ensuring your visibility preferences are strictly enforced on the backend.
