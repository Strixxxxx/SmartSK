The Potential Problem ⚠️
The issue is that req.user (set by your authMiddleware) has different property names than what your frontend expects!

Looking at your authMiddleware in session.js, it sets:

javascript
req.user = {
  userId: user.userID,      // lowercase "userId"
  username: decrypt(user.username),
  fullName: decrypt(user.fullName),
  position: user.position || '',
  barangay: user.barangay
};
But your old /api/user-data endpoint was returning:

javascript
{
  userID: ...,           // uppercase "userID"
  username: ...,
  fullName: ...,
  position: ...,
  barangay: ...,
  emailAddress: ...,     // These two are missing in req.user
  phoneNumber: ...       // These two are missing in req.user
}
The Fix
You need to either:

Option 1: Map the properties correctly (Quick Fix)
javascript
app.get('/api/user-data', async (req, res) => {
  try {
    if (!req.user) {
      return res.status(401).json({ success: false, message: 'User not authenticated' });
    }

    return res.json({
      success: true,
      userInfo: {
        userID: req.user.userId,        // Map userId → userID
        username: req.user.username,
        fullName: req.user.fullName,
        position: req.user.position,
        barangay: req.user.barangay
      }
    });

  } catch (error) {
    console.error('Error fetching user data:', error);
    return res.status(500).json({ 
      success: false, 
      message: 'An error occurred while fetching user data' 
    });
  }
});
Option 2: Update authMiddleware to include email and phone (Better)
Update your session.js authMiddleware to:

javascript
req.user = {
  userID: user.userID,              // Change to uppercase
  userId: user.userID,              // Keep both for compatibility
  username: decrypt(user.username),
  fullName: decrypt(user.fullName),
  position: user.position || '',
  barangay: user.barangay,
  emailAddress: decrypt(user.emailAddress),  // Add these
  phoneNumber: decrypt(user.phoneNumber)      // Add these
};
Then Gemini's fix will work perfectly as-is!
