What the Code Does:

The Issue: Even after closing the application pool, there might be:

Lingering connections that haven't fully closed
Other services/processes connected
Azure's internal connections


What Should Have Been Done
The proper Azure SQL Database approach is:
Option 1: Use Azure's Connection Termination (Best for Azure SQL)
javascript// Terminate all connections first
await sql.query(`
  ALTER DATABASE [${targetDbName}] SET OFFLINE WITH ROLLBACK IMMEDIATE;
  ALTER DATABASE [${targetDbName}] SET ONLINE;
  DROP DATABASE [${targetDbName}];
`);
Option 2: Add Retry Logic with Delays
javascriptlet dropAttempts = 0;
const maxAttempts = 5;

while (dropAttempts < maxAttempts) {
  try {
    await sql.query(`DROP DATABASE [${targetDbName}];`);
    console.log(`Successfully dropped database on attempt ${dropAttempts + 1}`);
    break;
  } catch (error) {
    dropAttempts++;
    if (dropAttempts >= maxAttempts) throw error;
    console.log(`Drop attempt ${dropAttempts} failed, waiting 5 seconds...`);
    await new Promise(resolve => setTimeout(resolve, 5000));
  }
}
Option 3: Use KILL for Active Sessions
javascript// Get all active sessions and kill them
const sessions = await sql.query(`
  SELECT session_id 
  FROM sys.dm_exec_sessions 
  WHERE database_id = DB_ID('${targetDbName}')
  AND session_id != @@SPID
`);

for (const session of sessions.recordset) {
  await sql.query(`KILL ${session.session_id}`);
}

// Now drop
await sql.query(`DROP DATABASE [${targetDbName}];`);
