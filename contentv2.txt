Great question! Yes, we absolutely can (and should) specify the backup redundancy for the temporary database in your restore script.

How to Specify Backup Redundancy in sqlpackage
When using sqlpackage /a:Import, you can add the /p:DatabaseBackupStorageRedundancy parameter:

bash
sqlpackage /a:Import \
  /tsn:server \
  /tdn:temp_database \
  /sf:backup.bacpac \
  /tu:user \
  /tp:password \
  /p:DatabaseEdition=Basic \
  /p:DatabaseMaximumSize=2 \
  /p:DatabaseBackupStorageRedundancy=Local  # ← ADD THIS
