✦ Of course. Using the Azure Portal is a good approach. Here is a step-by-step guide to the entire process.

  Important: This procedure involves creating and deleting Azure resources and will reconfigure your database's replication. While generally safe, it's best to perform this
  during a low-traffic period for your application.

  The goal is to force your database's metadata to reset by adding it to a new Failover Group and then immediately removing it.

  ---

  Part 1: Create a Temporary Partner SQL Server

  A failover group requires a second SQL server in a different Azure region. We will create a temporary one for this process.

   1. In the Azure Portal, click + Create a resource at the top left.
   2. Search for and select SQL Server. Click Create.
   3. Fill out the form:
       * Subscription: Choose your subscription.
       * Resource group: Select the same resource group as your smart-sk server (Smart_SK).
       * Server name: Give it a globally unique name (e.g., smart-sk-temp-partner- followed by some random numbers).
       * Location: Choose a different region from your primary server. Your primary is in southeastasia, so choose East Asia.
       * Authentication method: Select Use SQL authentication.
       * Server admin login: Create a temporary admin username (e.g., tempadmin).
       * Password: Create and confirm a strong temporary password.
   4. Click Review + create, and then Create. Wait for the new server to finish deploying.

  ---

  Part 2: Create the Failover Group

  Now we will add your smartSK database to a new failover group with the temporary server.

   1. Navigate to your original SQL server resource, `smart-sk`.
   2. In the left-hand menu, scroll down to the Data management section and click on Failover groups.
   3. At the top, click + Add group.
   4. On the creation page:
       * Failover Group name: Enter a temporary name, for example: temp-cleanup-fg.
       * Partner server: Select the temporary server you created in Part 1 (e.g., smart-sk-temp-partner-...).
       * Scroll down to Databases within the group. Below the list, click on the 1 database(s) on this server are not in any failover group text.
       * A new blade will open. Check the box next to your `smartSK` database and click Select.
   5. Click Create. The deployment will now start. This process may take several minutes as Azure creates a replica of your database on the partner server.

  ---

  Part 3: Delete the Failover Group (The Fix)

  This is the most important step. Once the group is created, we immediately delete it to clean the metadata.

   1. After the deployment in Part 2 is complete, you should be on the Failover groups page for your smart-sk server. You will see your new group (temp-cleanup-fg) in the list.
   2. Click on the name of the failover group you just created.
   3. On the group's overview page, click the Delete button at the top.
   4. Type the name of the failover group to confirm and click Delete.

  This will remove the replication link. The original error should now be resolved.
